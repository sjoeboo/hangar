Decision 3 of 4

Filter Bar Redesign

Visual

The current filter bar shows All ●1 ◐2 ○0 !@#$ filter · 0 all — status counts you can click, plus a hint that !/@/#/$ are keyboard shortcuts. Nobody reading that cold knows what !@#$ means. Now that we're adding view tabs above this row, we also need to make sure this sub-filter row clearly belongs to the Sessions view and doesn't visually compete with the tabs. How should we fix it?

Labeled Status Pills

Sessions · PRs · Todos
All ● Running ! ◐ Waiting @ ○ Idle #
SESSIONS
●feature-auth
◐fix-login-bug
○data-pipeline
Each filter pill gets a text label — Running, Waiting, Idle — so you immediately know what you're filtering to. The keyboard shortcut key (!, @, #) appears as a tiny superscript inside the pill. The !@#$ filter · 0 all hint is removed entirely. Click or key to activate; the active pill highlights in that status's color.

Works well when

  • New users need to understand what filtering does
  • Mouse users want to click a clearly-labeled target
  • Keyboard shortcuts still work but are learnable in context

Watch out for

  • Takes slightly more horizontal space than current
  • Labels can get crowded if terminal is narrow

Fix the Hint Text Only

All ● 1 ◐ 2 ○ 0 click or !/@ to filter
SESSIONS
●feature-auth
◐fix-login-bug
○data-pipeline
Keep the existing icon-count pills (● 1, ◐ 2, ○ 0) but replace the cryptic !@#$ filter · 0 all with plain English: click or !/@ to filter. Minimal code change, immediately more readable. The session counts stay compact. Mouse users can still click the pills.

Works well when

  • You want the smallest possible change
  • Existing users already know the icon meanings
  • Keeps the compact, information-dense style

Watch out for

  • Icons alone (●, ◐, ○) still require prior knowledge
  • Doesn't fully solve discoverability for new users

Remove the Filter Bar

SESSIONS ● 1 running · ◐ 2 waiting
●feature-auth
◐fix-login-bug

○refactor-api
Drop the filter bar entirely. Status counts move into the section header as a compact inline summary (SESSIONS ● 1 running · ◐ 2 waiting). Filtering by status gets removed — or stays as hidden keyboard-only feature without the bar. This gives back one full row of vertical space for sessions.

Works well when

  • You rarely actually filter by status
  • Recovering vertical space matters
  • Status counts in the header row are enough context

Watch out for

  • Loses a genuinely useful feature (filter to "show me just waiting sessions")
  • Keyboard shortcuts become truly hidden/undiscoverable
