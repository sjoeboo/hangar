Decision 2 of 4

Tab Visual Style

Visual

We've decided on an extensible 3-tab nav (Sessions, PRs, Todos). Now we need to decide how those tabs look. The hangar TUI already has a visual language — the oasis_lagoon_dark theme with rounded powerline pills in the tmux tab bar. The tab style should feel cohesive with what's already there while being clearly interactive. These are terminal renderings, so we're working with colored text, backgrounds, and unicode — not pixel-perfect icons.

Rounded pill buttons match the existing tmux window tabs in the status bar — so the whole UI feels cohesive. The active tab gets the cyan background, inactive tabs sit in a dark recessed state. This is the same visual language hangar already uses; adding these tabs just extends the pattern upward.

Works well when

  • Consistency with existing tmux tab styling matters
  • Users already associate pill = interactive element
  • Renders cleanly at all terminal font sizes

Watch out for

  • Could be confused with the status filter pills below it
  • May need a visual separator between tab row and filter row

The active tab gets an underline in cyan — the same style used by lazygit, delta, and many modern TUI tools. Inactive tabs are dimmed text. This style uses very little vertical space and creates clear visual hierarchy without colored backgrounds. The cyan underline connects naturally to the oasis theme's accent color.

Works well when

  • You want maximum vertical space for content
  • Familiar to lazygit users (popular TUI)
  • Clearly distinct from the pill-style filter row below

Watch out for

  • Underlines can be subtle in some terminal fonts
  • Less consistent with existing pill-style tmux tabs

A full-width colored header band holds the tabs. The band uses a deep blue background that makes the entire tab row visually prominent — you can't miss it. The active tab gets a brighter highlight within the band. This is visually striking but uses more color than the rest of hangar's UI.

Works well when

  • Maximum discoverability is the priority
  • You want tabs to feel like a distinct navigation zone

Watch out for

  • Heaviest visual weight — competes with content
  • May clash with oasis_lagoon_dark palette
  • Uses more vertical space than other options

Each tab shows its number shortcut badge (1, 2, 3) right on the tab itself, teaching users the keyboard shortcut just by looking at it. Press 1 for Sessions, 2 for PRs, 3 for Todos. This helps discoverability of both the views and their shortcuts simultaneously.

Works well when

  • You want users to learn keyboard shortcuts naturally
  • Self-documenting UI (no separate help needed)
  • Good balance of keyboard + mouse support

Watch out for

  • Badges add visual noise if you have many tabs
  • Number keys may conflict with other bindings
  • Confusing if status filter also uses number-adjacent keys (!@#)
